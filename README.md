## 📌깃 컨벤션
### 💡 커밋룰
#### Commit 메세지 작성 법
```
Type: 커밋내용 [지라이슈번호]
- Type은 영어로, 그 뒤 커밋 내용은 한글로 적기로 한다.
- ex) FIX : 로그인 에러 수정
```
#### Type 유형
```
FEAT: 새로운 기능 추가
FIX: 버그 수정
DOC: 문서 수정
CHORE: 그 외 자잘한 작업
TEST: 테스트 코드
BUILD: 시스템 또는 외부 종속성에 영향을 미치는 변경사항(npm, yarn 등)
CI: CI 관련 설정
STYLE: 코드 의미에 영향을 주지 않는 변경사항 (포맷, 세미콜론 누락, 공백 등)
REFACTOR: 성능 개선
MERGE: 기능 개발 후 브랜치 병합 
```
